nftables 모드 (kube-proxy)

개요

- nftables는 iptables의 차세대 대체 프레임워크로, 단일 계층의 표현식과 내부적인 객체(맵, 셋)를 사용해 더 효율적이고 원자적인 규칙 관리를 제공한다.
- kube-proxy가 nftables 모드로 동작하면 nft 규칙과 셋을 사용해 서비스 트래픽을 처리한다.

장점

- 현대적 설계: 규칙 업데이트의 원자성(atomic update), 표현식 기반의 유연한 룰 작성이 가능.
- 규칙 규모·복잡도 관리: 맵/셋을 활용해 많은 엔드포인트를 작은 객체로 효율적으로 관리할 수 있음.
- 퍼포먼스 이점: iptables보다 덜 반복적인 규칙 검사와 더 작은 규칙셋으로 성능 개선 가능.
- 일관된 API: nft 명령어로 IPv4/IPv6/ARP 등을 통합 관리.

단점

- 호환성/지원성: 상대적으로 최신 커널과 유틸리티 필요—옛 배포판에서는 제한적일 수 있음.
- 도구 친숙도: iptables에 익숙한 운영자에게는 새로운 문법·디버깅 방법 학습 필요.
- kube-proxy 구현 상태: kube-proxy의 nftables 지원은 배포판·버전별로 차이가 있을 수 있으니 안정성 확인 권장.

운영 팁

- 최신 리눅스 배포판에서는 nftables가 기본이므로 채택 시 운영상 이점이 큼.
- nftables의 set과 map을 적극 활용하면 룰 갱신 비용을 줄일 수 있음.
- 기존 iptables 기반 운영 환경에서는 전환 테스트(롤백 플랜 포함)를 권장.
